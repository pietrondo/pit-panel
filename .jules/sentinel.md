## 2026-07-13 - [Escape Command Output in System Manage]
**Vulnerability:** System manage commands (`system_manage.py:system_manage_action`) output (`output`) is directly rendered in an HTMLResponse without HTML escaping.
**Learning:** Returning unescaped output from external commands (`run_sudo`) creates a Cross-Site Scripting (XSS) vulnerability if the command output can be influenced by attackers or contains unexpected HTML characters.
**Prevention:** Always use `html.escape` to sanitize the output of system commands before returning it in an HTMLResponse.
## 2026-07-18 - [Fix XSS Vulnerabilities in Unescaped HTMLResponses]
**Vulnerability:** Raw variables like stdout and stderr generated by CLI wrappers or scanner outputs were directly injected into HTMLResponses for `wordpress.py` and `security_malware.py`.
**Learning:** Returning unescaped dynamic outputs from external commands without escaping can inadvertently result in Cross-Site Scripting (XSS) if the contents of those exceptions or commands contain executable HTML code tags.
**Prevention:** Explicitly apply `html.escape(str(variable))` onto all dynamically rendered strings inside endpoints passing variables via `HTMLResponse`.
