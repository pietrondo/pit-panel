## 2025-07-05 - Fix Event Loop Blocking DoS in Subprocess Calls
**Vulnerability:** Synchronous `subprocess.run()` calls were used to execute long-running tasks (like `git clone`, system upgrades, and `fail2ban-client`) directly within FastAPI route handlers. This caused the asyncio event loop to block entirely, creating a Denial of Service (DoS) vulnerability where no other concurrent requests could be processed until the shell command finished.
**Learning:** Even if command arguments are passed as a list and prevent direct shell injection, using synchronous I/O operations (like `subprocess.run`) in an async web framework is inherently unsafe because it starves the event loop. Furthermore, when migrating to `asyncio.create_subprocess_exec` inside a `try...except` block, calling `proc.kill()` in the exception handler without checking if `proc` was initialized can lead to `UnboundLocalError`.
**Prevention:** Always use `asyncio.create_subprocess_exec` combined with `asyncio.wait_for(proc.communicate(), timeout=X)` for executing external commands within ASGI applications. Additionally, when handling timeouts and process termination, use `if 'proc' in locals(): proc.kill()` or pre-initialize `proc = None` to gracefully handle early creation failures.
